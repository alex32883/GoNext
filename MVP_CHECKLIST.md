# MVP Checklist — GoNext

Проверка сценариев использования и офлайн-режима.

---

## Офлайн-режим

Приложение работает **без интернета**:

- ✅ SQLite — все данные локально
- ✅ Фотографии — в файловой системе приложения
- ✅ Навигация по экранам — без сети
- ℹ️ Кнопки «Открыть на карте» / «В навигатор» открывают системные приложения (могут требовать интернет для загрузки карт)

---

## Сценарий: добавление мест → поездка → следующее место

1. **Добавить места**
   - Места → FAB «+» → заполнить название, описание, координаты (широта/долгота) → Сохранить
   - Добавить 2–3 места с координатами

2. **Создать поездку**
   - Поездки → FAB «+» → название, даты, отметить «Текущая поездка» → Создать

3. **Добавить места в поездку**
   - Открыть созданную поездку → «Добавить место» → выбрать место из списка
   - Добавить несколько мест, изменить порядок (↑↓)

4. **Отметить посещения**
   - В маршруте отметить первое место как посещённое (чекбокс)
   - Добавить заметки, фото (иконка камеры)

5. **Следующее место**
   - Следующее место → должно показаться следующее непосещённое место
   - Кнопки «Открыть на карте» и «В навигатор»
   - После отметки посещения — экран обновится при возврате (useFocusEffect)

---

## Сообщения пользователю

- **Нет текущей поездки** — показывается на экране «Следующее место» с кнопкой «Перейти в Поездки»
- **Все места посещены** — «Все места в текущей поездке посещены. Отличная работа!»
- **Нет координат** — «У места «X» не указаны координаты. Добавьте их в карточке места.»

## Навигация (PROJECT.md)

- Главный экран: GoNext, кнопки Места / Поездки / Следующее место / Настройки
- Список мест, карточка места, добавление места
- Список поездок, детали поездки, создание поездки
- Экран «Следующее место»
- Настройки (версия, информация об офлайн-режиме)

Без вложенных сложных меню.
